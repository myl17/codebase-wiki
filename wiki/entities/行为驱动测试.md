# 行为驱动测试

测试公共 API 表面和可观察行为，而非内部实现细节。避免对私有方法或 mock 内部调用做断言，使测试在重构时保持稳定。

## 在各 repo 中的体现

- [[openclaw/dimensions/testing-philosophy]] — 大量 `vi.mock` 用于隔离外部依赖，但测试目标是行为和边界条件而非实现路径
- [[hermes-agent/dimensions/testing-philosophy]] — 明确原则：测试公共 API 表面、错误路径和边界值；不测私有实现细节，不对 mock 对象的内部调用过度断言
