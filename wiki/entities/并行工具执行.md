# 并行工具执行

LLM agent 在单轮 tool-calling 中同时执行多个无依赖关系的工具调用，以降低总延迟。需区分哪些工具可安全并行（只读、无共享状态）、哪些必须串行（交互式、有副作用）。

## 在各 repo 中的体现

- [[hermes-agent/dimensions/hermes-agent-performance-tradeoffs]] — 最大 8 worker 线程；永不并行：`clarify` 等交互工具；安全并行：`web_search`、`read_file` 等 11 个只读工具
