# 插件系统

通过注册 API 或声明式 manifest 将核心运行时与可扩展能力解耦：核心不感知具体实现，扩展包按需加载，故障域互不影响。

## 在各 repo 中的体现

- [[openclaw/dimensions/openclaw-extension-points]] — 双层设计：声明式 `openclaw.plugin.json` + 命令式 `OpenClawPluginApi`；`definePluginEntry` / `defineBundledChannelEntry` 为统一入口；channel 按需懒加载
- [[openclaw/dimensions/openclaw-architecture]] — `ChannelPlugin<ResolvedAccount>` 接口统一 13+ 个 adapter；`extensions/` 下每个目录是独立 npm 包，20+ 平台互相隔离
- [[hermes-agent/dimensions/hermes-agent-extension-points]] — Tool Registry 通过 AST 扫描自动发现 `registry.register()` 调用；Toolset System 可组合；Memory/ContextEngine/Platform Adapter 各自独立扩展点
